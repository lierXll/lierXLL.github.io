Spring AOP的用法

发表于 2020-03-13 | 分类于 java ， spring

AOP：面向切面编程

基于通过预编译方式和运行期间动态代理实现，使业务逻辑各部分之间的耦合度降低。

在运行时，动态地将代码切入到类的指定方法、指定位置上的编程思想就是面向切面的编程。

比如，声明式事务、日志、安全、缓存等等，都与业务逻辑无关，可以将这些东西抽象成为模块，采用面向切面编程的方式，通过声明方式定义这些功能用于何处，通过预编译方式和运行期动态代理实现这些模块化的程序功能进行统一维护，从而将这些模块与它们所影响的对象之间分离出来，就是实现解耦。

AOP的术语

切面（Aspect）：切入到指定类指定方法的代码片段称为切面
切入点（Pointcut）：切入到哪些类、哪些方法叫切入点（切点定义了切面”去何处做”）
5类通知（Advice)（通知定义的是切面“做什么”和“在何时做”）
前置通知（Before Advice）：在切入点方法执行之前，插入的通知；
后置通知（After Advice）：在切入点方法执行完毕，插入的通知；
异常置通知（After Trowing Advice）：在切入点方法抛出异常时，插入的通知；
最终通知（Before FinallyAdvice）：在切入点方法执行完毕（不论正常返回或异常退出），插入的通知；
环绕通知（Around Advice）：可以贯穿切入点方法执行的整个过程

一、什么是反射

Java反射机制是在运行状态中，对于任意一个类，都能够知道这个类的所有属性和方法；对于任意一个对象，都能够调用它的任意一个方法和属性；这种动态获取的信息以及动态调用对象的方法的功能称为Java语言的反射机制。

安装Hexo:

1.安装node.js

检查node是否安装成功: node -v
检查npm是否安装成功: npm -v

2.安装git

检查是否git安装成功: git –version

3.使用npm安装Hexo

安装命令: npm install -g hexo-cli
检查npm是否安装成功: hexo -v
初始化: hexo init
安装生成器: npm install
安装hexo git插件: npm install hexo-deployer-git –save
新建一篇博客,目录为D:\BLOG\source_posts: hexo new “文件名称”
生成网页: hexo g
部署到本地，localhost:4000访问: hexo s
清理缓存: hexo clean
部署到: github: hexo d
创建(首页)文件夹,目录为D:\BLOG\source\home，并生成index.md: hexo new ‘home’
